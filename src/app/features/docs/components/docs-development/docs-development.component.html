РАЗРАБОТКА ПРИЛОЖЕНИЯ Разработка диаграммы прецедентов

Диаграмма прецедента поможет описать функциональное назначение система или то, что система должна делать. Проектируемая система представлена в виде множества сущностей или актеров, взаимодействующих с системой с помощью прецедентов. При этом (actor) или действующим лицом называется любая сущность, взаимодействующая с системой извне. Это может быть человек, техническое устройство, программа или любая другая система, которая может служить источником воздействия на систему так, как определит разработчик, прецедент служит для описания сервисов, которые система предоставляет актеру.

Разработка кода приложения

Приложение состоит из 2-ух частей: клиентской и серверной.

В свою очередь клиентских приложений может быть несколько, на данный момент их 2: веб-клиент и андроид-клиент.

Серверная часть состоит из 2-ух проектов: WebSocketServer - сервер для обмена сообщениями в реальном времени и RestApiServer - сервер, для базовых CRUD-операций

Тестирование программы

Следующим пунктом выполнения проекта стало тестирование работоспособности готового приложения «Web-чат». Существует несколько стратегий тестирования приложения, в том числе черного ящика.

В данной стратегии приложение рассматривается как чёрный ящик. Целью тестирования ставится выяснение обстоятельств, в которых поведение приложения не соответствует спецификации. Для обнаружения всех ошибок в приложении необходимо выполнить исчерпывающее тестирование, то есть тестирование на всевозможных наборах данных. Для большинства приложений такое невозможно, поэтому применяют разумное тестирование, при котором тестирование программы ограничивается небольшим подмножеством всевозможных наборов данных. При этом необходимо выбирать наиболее подходящие подмножества, подмножества с наивысшей вероятностью обнаружения ошибок. В качестве исполняемой ветки программы было решено протестировать регистрацию пользователя.

Руководство программиста:

Все необходимые инструкции описаны в файлах INSTALL.md, README.md для каждого из проектов.

Назначение и условия применения программы: программа должна обеспечивать поддержку следующих возможностей обработки информации:

  • предоставить возможность вводить данные с клавиатуры;
  • регистрировать пользователя;
  • авторизовать пользователя по логину и паролю;
  • предоставить пользователю доступ к обмену сообщениями в общем чате;
  • предоставить пользователю доступ к личным сообщениям.

Руководство пользователя:

Характеристика приложения: для работы с приложением необходимо обладать минимальными знаниями. Простой и интуитивно понятный интерфейс позволяет работать с приложением любому пользователю, имеющему начальные навыки общения с компьютером. Приложение поддерживает интерфейс пользователя на основе меню, ориентированный на использование клавиатуры и манипулятора-мышь.

Порядок выполнения приложения, после того, как сервер успешно запущен:

  • Если пользователь не зарегистрирован, то надо пройти регистрацию;
  • Если пользователь уже зарегистрирован, то авторизоваться по логину и паролю;
  • авторизовать пользователя по логину и паролю;
  • Приступить к обмену сообщениями между пользователями чата.

После перехода на сайт, пройдем регистрацию по кнопке «SIGN UP», от пользователя требуется указать логин и пароль, и выбрать один из предложенных аватаров.
